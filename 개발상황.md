## 프로젝트 요약: 현재 상태

현재 시스템은 **"프론트엔드 - 백엔드 - 워커"의 전체 파이프라인이 로컬에서 정상 기동**되는 상태입니다. 사용자가 도서를 검색하고 영상 생성을 요청하면, 워커가 이를 수신하여 AI 엔진(Pipeline V2)을 통해 처리를 시작하는 전 과정의 인터페이스가 구축되었습니다.

| 구분 | 상태 | 비고 |
| --- | --- | --- |
| **인프라 통합** | ✅ 완료 | Trailer-Engine의 핵심 엔진을 Worker 패키지로 병합 완료 |
| **API 통신** | ✅ 완료 | 영상 생성 요청 ↔ 결과 콜백(Ready/Failed) 연동 완료 |
| **안전 검증** | ✅ 완료 | 어린이 도서관 기준에 맞춘 키워드 기반 콘텐츠 필터링 적용 |
| **AI 파이프라인** | 🔄 진행중 | Pipeline V2 진입점 전환 완료 (실제 영상 합성 단계 연동 중) |
| **실행 환경** | ✅ 정리 | Backend Fastify 5 + 플러그인 호환 완료, Worker .env 로드 경로 수정 |
| **프론트 DID/프로토타입** | ✅ 반영 | 영상 URL 수정, /pip 라우트·fallback 샘플 영상 추가 |

* **완료 일시:** 2026-02-09

## 테스트 가이드 (협업용)

디자이너 또는 테스트 인원이 로컬에서 시스템을 확인하기 위한 최소 설정입니다.

### 1. 환경 변수 설정 (.env)

루트 폴더에 아래 설정이 필요합니다.

```env
GEMINI_API_KEY=실제_키_입력
INTERNAL_API_SECRET=임의의_문자열
DATABASE_URL=file:./packages/backend/prisma/dev.db
REDIS_HOST=localhost

```

### 2. 실행 명령어 순서

```bash
# 1. 의존성 설치 및 DB 초기화
npm install
npm run db:generate

# 2. 시스템 기동 (프론트/백/워커 동시 실행)
npm run dev

```
