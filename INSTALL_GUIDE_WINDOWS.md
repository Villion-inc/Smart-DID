# 꿈샘 도서관 스마트 DID - 로컬 설치 및 테스트 가이드

---

## 1. 시스템 구조 요약

```
[도서관 PC - 로컬 실행]                       [GCP 클라우드]

  Backend (API 서버)  ─── 인터넷 ───>  Cloud SQL (PostgreSQL DB)
  Frontend (웹 화면)  ─── 인터넷 ───>  Cloud Storage (영상 파일)
       |                              Cloud Run Worker (영상 생성)
       v                              Cloud Memorystore (Redis 큐)
  ALPAS 도서관 서버
  (내부망 10.10.11.13)
```

### 로컬에서 하는 것

| 항목 | 설명 |
|------|------|
| 도서 검색 | ALPAS 내부망 서버에서 실시간 검색 |
| 신착도서 목록 | ALPAS 내부망 서버에서 조회 |
| DID 키오스크 화면 | 프론트엔드 웹 화면 표시 |
| 영상 재생 | 이미 생성된 영상을 클라우드에서 불러와 재생 |

### 클라우드에서 하는 것

| 항목 | 설명 |
|------|------|
| 데이터베이스 | 영상 상태, 추천도서 등 모든 데이터 저장 |
| 영상 생성 | Admin 페이지에서 요청 시 자동 생성 |
| 영상 저장 | 생성된 영상 파일을 Cloud Storage에 저장 |

### 로컬에 설치하지 않는 것

- PostgreSQL (데이터베이스) - 클라우드 DB 사용
- Redis (영상 생성 큐) - 클라우드에서만 사용
- Worker (영상 생성기) - 클라우드에서만 실행

## 3. 사전 준비 (도서관 방문 전)

도서관 PC에서 클라우드 데이터베이스에 접근하려면, 도서관의 공인 IP를 GCP에 등록해야 합니다.

### 3-1. 도서관 공인 IP 확인

도서관 PC에서 브라우저를 열고 아래 주소에 접속합니다:

```
https://whatismyip.com
```

화면에 표시되는 IP 주소를 메모합니다. (예: `211.234.56.78`)

### 3-2. GCP Cloud SQL에 IP 등록

**방법 A: GCP 웹 콘솔 (권장)**

1. 브라우저에서 https://console.cloud.google.com 접속
2. 상단에서 프로젝트 `asanlibrary` 선택
3. 좌측 햄버거 메뉴 > **SQL** 클릭
4. 인스턴스 목록에서 **smart-did** 클릭
5. 상단 **연결** 탭 클릭
6. **승인된 네트워크** 섹션까지 스크롤
7. **네트워크 추가** 버튼 클릭
8. 이름: `library` 입력
9. 네트워크: `도서관IP/32` 입력 (예: `211.234.56.78/32`)
10. **완료** 클릭 후 **저장** 클릭
11. 적용까지 1~2분 대기

**방법 B: 빠른 테스트용 (임시 전체 허용)**

테스트 시에만 모든 IP를 임시 허용하고, 테스트 후 도서관 IP만 남깁니다:

```bash
gcloud sql instances patch smart-did \
  --project=asanlibrary \
  --authorized-networks=0.0.0.0/0
```

> 주의: 테스트 완료 후 반드시 도서관 IP만 남기고 제거하세요.

## 4. 설치 및 테스트 순서

### 순서 요약

```
1단계: USB로 프로젝트 복사
          |
2단계: setup.bat 실행
          |
          +-- Node.js 설치
          +-- GCP 인증 (브라우저 Google 로그인, 최초 1회)
          +-- 환경변수 설정
          +-- ALPAS 연동 자동 테스트 --+
          |                           |
       성공 시                     실패 시
          |                           |
          +-- 의존성/빌드          클라우드 베타 버전 자동 열림
          |                       (추가 설치 불필요)
3단계: start-all.bat 실행
          |
4단계: 동작 확인
```

---

### 4-1. 1단계: 프로젝트 복사

USB 메모리에서 `lib-mvp` 폴더 전체를 도서관 PC에 복사합니다.

- 권장 위치: `C:\smart-did\lib-mvp`
- 경로에 한글이나 공백이 없는 것을 권장합니다.

복사 후 폴더 구조:
```
C:\smart-did\lib-mvp\
  ├── scripts\          <-- 여기에 실행 파일들
  │   ├── setup.bat
  │   ├── start-all.bat
  │   └── test-alpas.bat
  ├── library-did-main\ <-- 프로젝트 소스
  └── ...
```

---

### 4-2. 2단계: 설치

`scripts\setup.bat` 파일을 **더블클릭**합니다.

> "관리자 권한" 확인 창이 뜨면 **예**를 클릭하세요.

setup.bat이 자동으로 처리하는 것들:

| 순서 | 작업 | 설명 |
|------|------|------|
| 1 | Node.js 설치 | 이미 설치되어 있으면 건너뜀 |
| 2 | GCP 인증 | 브라우저에서 Google 로그인 (최초 1회, 영상 접근용) |
| 3 | 환경변수 설정 | .env 파일 생성 및 ALPAS 설정 확인 |
| 4 | ALPAS 연동 테스트 | 내부망 ALPAS 서버 연결 확인 |
| 5 | 의존성 설치 | npm 패키지 (약 3~5분) |
| 6 | DB 연결 | GCP Cloud SQL 연결 확인 |
| 7 | 빌드 | 백엔드/프론트엔드 서버 빌드 |

**GCP 인증 (최초 1회)**

설치 중 브라우저가 자동으로 열립니다.
Google 계정으로 로그인하면 영상 파일(Cloud Storage) 접근 권한이 설정됩니다.
이 인증은 한 번만 하면 되며, PC를 재시작해도 유지됩니다.

**중요: 환경변수 확인**

설치 중 메모장이 자동으로 열립니다. 아래 항목이 올바른지 확인하세요:

```
ALPAS_API_URL=http://10.10.11.13:28280/METIS/HOMEPAGE/API
ALPAS_API_KEY=1
ALPAS_LIB_NO=144045
ALPAS_MANAGE_CODE=CH
```

확인 후 메모장을 저장하고 닫은 뒤, 설치 창에서 아무 키나 누르면 계속 진행됩니다.

**ALPAS 연동 테스트 결과:**

- **성공 시**: "설치 완료!" 메시지가 표시됩니다. 3단계로 진행하세요.
- **실패 시**: 자동으로 클라우드 베타 버전이 브라우저에서 열립니다. 로컬 설치는 중단되며, 클라우드 버전을 사용합니다.

설치 시간: 약 5~10분 (인터넷 속도에 따라 다름)

---

### 4-3. 3단계: 실행

`scripts\start-all.bat` 파일을 **더블클릭**합니다.

두 개의 검은 창(터미널)이 열리면서 서버가 시작됩니다:

| 서비스 | 포트 | 창 제목 |
|--------|------|---------|
| 백엔드 API 서버 | 3001 | Smart DID Backend |
| 프론트엔드 웹 | 5173 | Smart DID Frontend |

약 5초 후 브라우저가 자동으로 열립니다.

> "QueueService: REDIS_HOST not configured, queue disabled" 경고가
> 백엔드 창에 표시될 수 있습니다. 이것은 정상입니다.
> 영상 생성 큐는 클라우드에서만 사용되므로 무시하세요.

---

### 4-4. 4단계: 동작 확인

브라우저에서 아래 항목을 하나씩 확인합니다.

| # | 확인 항목 | 방법 | 예상 결과 |
|---|----------|------|-----------|
| 1 | DID 메인 화면 | http://localhost:5173/did 접속 | 추천도서 목록이 표시됨 |
| 2 | 도서 검색 | 검색 버튼으로 "동화" 검색 | 검색 결과가 표시됨 |
| 3 | 신착도서 | 신작 버튼 클릭 | 신착도서 목록이 표시됨 |
| 4 | 도서 상세 | 도서 하나를 터치/클릭 | 제목, 저자, 줄거리 표시 |
| 5 | 영상 재생 | 영상이 있는 도서 상세에서 확인 | 영상 자동 재생 |
| 6 | 관리자 페이지 | http://localhost:5173/admin 접속 | 클라우드 Admin으로 자동 이동 |

## 5. ALPAS 연동 실패 시 (클라우드 베타 버전)

ALPAS 내부망 서버에 연결할 수 없는 경우, 자동으로 클라우드 베타 버전이 브라우저에서 열립니다.

클라우드 베타 URL:
```
https://did-frontend-730268485621.asia-northeast3.run.app
```

이 경우:
- 별도 로컬 설치가 필요하지 않습니다
- 위 URL을 브라우저에서 바로 사용합니다
- 검색/신착 기능은 ALPAS 공개 테스트 서버 데이터를 사용합니다
- 영상 생성은 클라우드 Admin에서 동일하게 사용 가능합니다

## 6. 영상 생성 방법 (관리자 전용)

영상 생성은 DID 키오스크에서 하지 않습니다.
**클라우드 Admin 페이지에서만** 진행합니다.

> **참고:** 로컬에서 `http://localhost:5173/admin`에 접속하면
> 자동으로 클라우드 Admin 페이지로 이동합니다.

### 영상 생성 절차

1. 클라우드 Admin 페이지에 접속하여 로그인 (로컬에서 /admin 접속 시 자동 이동)
2. **추천도서 등록** 메뉴에서 제목/저자 입력
3. 영상이 자동으로 생성됩니다 (약 2~5분 소요)
4. 생성 완료 후 DID 키오스크에서 해당 도서를 선택하면 영상이 재생됩니다

### 영상 생성 상태 확인

Admin > 영상 관리 메뉴에서 확인:

| 상태 | 의미 |
|------|------|
| 대기중 (QUEUED) | 생성 대기열에 등록됨 |
| 생성중 (GENERATING) | 현재 영상을 만들고 있음 |
| 완료 (READY) | 영상 생성 완료, DID에서 재생 가능 |
| 실패 (FAILED) | 생성 실패, 재시도 가능 |

## 7. 키오스크 자동 시작 설정 (선택)

PC 부팅 시 자동으로 서버와 브라우저가 시작되게 하려면:

### 7-1. 서버 자동 시작

1. 키보드에서 `Win + R` 키를 동시에 누릅니다
2. `shell:startup` 을 입력하고 확인을 누릅니다
3. 시작 프로그램 폴더가 열립니다
4. `scripts\start-all.bat` 파일을 **우클릭 > 바로가기 만들기**
5. 만들어진 바로가기를 시작 프로그램 폴더에 복사합니다

### 7-2. 브라우저 전체화면 자동 시작

1. 바탕화면에서 Chrome 아이콘을 **우클릭 > 바로가기 만들기**
2. 만들어진 바로가기를 **우클릭 > 속성**
3. **대상(T)** 항목의 맨 끝에 아래 내용을 추가합니다:
   ```
    --kiosk http://localhost:5173
   ```
   (기존 내용 뒤에 한 칸 띄우고 추가)
4. 이 바로가기를 시작 프로그램 폴더(`shell:startup`)에 복사합니다

## 8. 문제 해결

### ALPAS API 연결 실패

증상: "ALPAS 서버에 연결할 수 없습니다" 메시지

확인 사항:
1. 도서관 내부 네트워크(LAN)에 연결되어 있는지 확인
2. 명령 프롬프트에서 `ping 10.10.11.13` 실행하여 응답 확인
3. `.env` 파일에서 `ALPAS_API_URL` 주소가 올바른지 확인
4. 브라우저에서 `http://10.10.11.13:28280/METIS/HOMEPAGE/API/AE117.do` 직접 접속 시도

### DB 연결 실패 ("Can't reach database server")

증상: 검색은 되지만 영상 상태 조회 시 오류

확인 사항:
1. 인터넷에 연결되어 있는지 확인
2. GCP Cloud SQL에 도서관 공인 IP가 허용되어 있는지 확인
3. 위 "3-2. GCP Cloud SQL에 IP 등록" 참고

### 영상이 재생되지 않음

확인 사항:
1. 해당 도서의 영상이 Admin에서 생성되었는지 확인
2. 영상 상태가 READY인지 확인
3. 인터넷 연결 확인 (영상 파일은 클라우드에서 다운로드)
4. GCP 인증이 되어 있는지 확인 (setup.bat에서 Google 로그인을 완료했는지)

### 서버가 시작되지 않음

확인 사항:
1. `setup.bat`을 먼저 실행했는지 확인
2. 포트 3001, 5173이 다른 프로그램에서 사용 중이 아닌지 확인
3. Windows 방화벽에서 해당 포트 허용
