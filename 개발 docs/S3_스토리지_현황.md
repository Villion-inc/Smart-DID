# S3 스토리지 실제 구현 — 현재까지 된 것 / 남은 것

---

## 1. 지금까지 된 것

### 1.1 추상화 구조

| 항목 | 상태 | 설명 |
|------|------|------|
| **StorageProvider 인터페이스** | ✅ 완료 | `storage.provider.ts`: `save(key, data, metadata?)`, `load(key)`, `exists(key)`, `getUrl(key)` |
| **LocalStorageProvider** | ✅ 완료 | `storage-local.provider.ts`: 로컬 디렉터리에 파일 쓰기/읽기, `getUrl`은 `/videos/{key}` 반환 |
| **getStorageProvider()** | ✅ 완료 | `storage-provider.factory.ts`: `config.storage.type`이 `s3`면 S3, 아니면 Local 반환 |

### 1.2 S3StorageProvider (스텁)

| 항목 | 상태 | 내용 |
|------|------|------|
| **클래스·인터페이스 구현** | ✅ 있음 | `StorageProvider` 구현, 메서드 시그니처 맞춤 |
| **환경변수 읽기** | ✅ 있음 | `S3_BUCKET`, `S3_REGION`, `S3_PUBLIC_BASE_URL` (없으면 `https://{bucket}.s3.{region}.amazonaws.com`) |
| **save()** | ❌ 스텁 | `throw new Error('S3StorageProvider not implemented; use STORAGE_TYPE=local')` + 주석 `// TODO: AWS SDK PutObject 연동` |
| **load()** | ❌ 스텁 | 동일하게 throw |
| **exists()** | ❌ 스텁 | 항상 `false` 반환 (bucket 없으면 false, 있어도 미구현이라 false) |
| **getUrl(key)** | ⚠️ 부분 | `baseUrl + "/" + key` 로 **퍼블릭 URL만** 반환. **Presigned URL 아님.** 버킷이 private면 이 URL로 접근 불가. |

## 2. 아직 안 된 것 (실제 구현 시 할 일)

| 항목 | 내용 |
|------|------|
| **AWS SDK 연동** | `@aws-sdk/client-s3` (v3) 설치 후, PutObject / GetObject / HeadObject 등 호출. |
| **save() 구현** | Buffer를 S3 `PutObject`로 업로드, 성공 시 반환할 URL 결정 (퍼블릭 경로 또는 나중에 presigned로 줄 키). |
| **load() 구현** | `GetObject`로 Body 스트림/Buffer 읽기. |
| **exists() 구현** | `HeadObject`로 존재 여부 확인. |
| **Presigned URL** | private 버킷이면 `getUrl()` 또는 별도 메서드에서 `getSignedUrl`(v2) 또는 `GetObjectCommand` + `getSignedUrl`(v3)로 일시적 다운로드/스트리밍 URL 생성. (보통 만료 시간 1시간 등 지정.) |
| **환경변수** | `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY` (또는 IAM 역할). 이미 .env.example에 S3 관련 항목 있음. |

## 3. 실제로 누가 쓰는지

| 호출처 | 사용 방식 |
|--------|-----------|
| **video-generator.service.ts** | **storageService** (로컬 전용 `StorageService`) 사용. `getStorageProvider()` 미사용. |
| **Pipeline V2 (orchestrator)** | Step 7에서 아직 파일 저장/URL 설정만 placeholder. StorageProvider 또는 storageService 미호출. |

**정리:**  
- **StorageProvider/S3 스텁**은 “구조만” 갖춰져 있고,  
- **실제 저장 경로**는 아직 **storageService**(로컬 경로 직접 쓰기) 또는 Pipeline V2 placeholder에 머물러 있음.  
- S3를 실제로 쓰려면:  
  1) **S3StorageProvider**에 AWS SDK로 save/load/exists/presigned 구현하고,  
  2) **최종 영상/자막을 저장하는 쪽**(Pipeline V2 Step 7 또는 공통 저장 레이어)에서 **getStorageProvider()**를 쓰도록 바꾸면 됨.

---

## 4. 요약

| 구분 | 상태 |
|------|------|
| **추상화(인터페이스·로컬·팩토리)** | ✅ 구현됨 |
| **S3StorageProvider** | ❌ 스텁만 (save/load/exists 미구현, getUrl은 퍼블릭 URL만) |
| **Presigned URL** | ❌ 미구현 |
| **실제 저장 경로에서 Provider 사용** | ⚠️ 아직 storageService 직접 사용·Pipeline placeholder라, S3 연동 후 “저장 담당”을 getStorageProvider()로 바꾸는 작업 필요 |

즉, **“S3 스토리지 실제 구현”**은 **지금 인터페이스·로컬·S3 스텁까지** 되어 있고, **AWS SDK 연동 + save/load/exists + (필요 시) presigned URL + 저장 호출부를 Provider로 통일**하면 됨.
