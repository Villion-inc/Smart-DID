# Smart DID — 테스트 준비 및 명령어

지금까지 작업한 **Backend + Worker + 콜백·안전검증·GeminiProvider** 등을 로컬에서 테스트하기 위한 준비와 명령어입니다.

---

## 1. 사전 준비 (한 번만)

### 1.1 Redis 실행

Worker(BullMQ)가 Redis를 사용합니다. **반드시 먼저 실행**해야 합니다.

```bash
# macOS (Homebrew)
brew services start redis

# 또는 Docker
docker run -d -p 6379:6379 --name redis redis:latest

# 또는 직접 실행 (포그라운드)
redis-server
```

확인:

```bash
redis-cli ping
# PONG 이 나오면 정상
```

### 1.2 환경변수 설정

`library-did-main` 루트에서:

```bash
cd library-did-main
cp .env.example .env
```

`.env`에서 **최소 아래 값은 반드시 설정**:

| 변수 | 설명 | 예시 |
|------|------|------|
| `DATABASE_URL` | Backend Prisma DB | `file:./packages/backend/prisma/dev.db` |
| `JWT_SECRET` | 로그인·인증 (32자 이상) | `your-secret-key-change-in-production-32chars` |
| `INTERNAL_API_SECRET` | Worker → Backend 콜백 인증 | `your-internal-secret-change-in-production` |
| `REDIS_HOST` | Redis 호스트 | `localhost` |
| `REDIS_PORT` | Redis 포트 | `6379` |
| `GEMINI_API_KEY` | Worker 스크립트/키프레임용 (**없으면 Worker 기동 시 에러**) | [Google AI Studio](https://aistudio.google.com/apikey)에서 발급 |
| `BACKEND_URL` | Worker가 콜백 호출할 Backend 주소 | `http://localhost:3000` |

**최소 .env 예시** (복사 후 키만 교체):

```env
DATABASE_URL=file:./packages/backend/prisma/dev.db
JWT_SECRET=your-secret-key-change-in-production-32chars
INTERNAL_API_SECRET=your-internal-secret-change-in-production
REDIS_HOST=localhost
REDIS_PORT=6379
GEMINI_API_KEY=여기에_실제_Gemini_API_키
BACKEND_URL=http://localhost:3000

PORT=3000
NODE_ENV=development
LOG_LEVEL=info
STORAGE_TYPE=local
STORAGE_PATH=./storage/videos
WORKER_CONCURRENCY=2
```

### 1.3 DB 마이그레이션

**현재 디렉터리가 `library-did-main`인 상태에서** (앞에서 `cd library-did-main` 이미 했다면 `packages/backend`만 이동):

```bash
cd packages/backend
npm run prisma:migrate:deploy
npm run prisma:generate
cd ../..
```

한 줄로 실행하려면 (`library-did-main` 루트에서):

```bash
cd packages/backend && npm run prisma:migrate:deploy && npm run prisma:generate && cd ../..
```

> ⚠️ **`npx prisma` 사용 금지.** 전역/최신 Prisma 7이 설치되면 스키마 문법이 달라져 에러가 납니다. 반드시 **`npm run prisma:...`** 로 프로젝트에 포함된 Prisma 5를 사용하세요.  
> ⚠️ 경로는 `library-did-main` 안에서는 **`packages/backend`** 만 사용합니다.

### 1.4 의존성 설치

**루트 (모든 패키지):**

```bash
cd library-did-main
npm install
```

**Worker 전용** (Gemini·FFmpeg 연동용 — package.json에 없으면 수동 설치):

```bash
cd packages/worker
npm install @google/generative-ai fluent-ffmpeg
npm install -D @types/fluent-ffmpeg
cd ../..
```

> **참고:** `gemini-client`, `video-processor` 등이 위 패키지를 사용하므로, 설치하지 않으면 Worker 기동 시 `Cannot find module '@google/generative-ai'` 등 에러가 납니다.

## 2. 실행 명령어

작업 디렉터리: **`library-did-main`** (프로젝트 루트)

### 2.1 전체 실행 (Backend + Frontend + Worker)

```bash
npm run dev
```

- **Backend API:** http://localhost:3000  
- **Frontend:** http://localhost:5173  
- **Worker:** 같은 터미널에서 백그라운드로 동작  

### 2.2 개별 실행 (디버깅 시)

터미널 3개를 열어 각각:

```bash
# 터미널 1 — Backend
npm run dev:backend

# 터미널 2 — Frontend
npm run dev:frontend

# 터미널 3 — Worker
npm run dev:worker
```

**실행 순서:** Backend → (선택) Frontend → Worker. Worker는 Backend가 떠 있어야 콜백을 보낼 수 있습니다.

## 3. 동작 확인

### 3.1 Backend 헬스체크

```bash
curl http://localhost:3000/health
```

예상 응답: `{"status":"ok","timestamp":"..."}`

### 3.2 프론트엔드

- 브라우저: http://localhost:5173  
- 관리자 로그인: http://localhost:5173/admin/login  
  - 계정: `.env`의 `ADMIN_USERNAME` / `ADMIN_PASSWORD` (기본: `admin` / `changeme123`)

### 3.3 영상 생성 플로우 (E2E 테스트)

1. http://localhost:5173 에서 도서 검색 (예: "별")
2. 도서 클릭 → **「영상 생성 요청」** 클릭
3. 상태 변화 확인: **QUEUED** → **GENERATING** → **READY** (또는 실패 시 **FAILED**)
4. Worker 로그에서 콜백 전송 및 완료/실패 메시지 확인

### 3.4 API로 직접 테스트

```bash
# 로그인
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"changeme123"}'
# 응답의 token 값을 복사

export TOKEN="여기에_JWT_토큰"

# 도서 검색
curl "http://localhost:3000/api/books?query=별"

# 영상 생성 요청 (실제 ISBN으로 교체)
curl -X POST "http://localhost:3000/api/books/ISBN-001/video" \
  -H "Authorization: Bearer $TOKEN"

# Admin: 사전 생성 (인증 필요)
curl -X POST "http://localhost:3000/api/admin/books/ISBN-002/video" \
  -H "Authorization: Bearer $TOKEN"
```

## 5. 선택 환경변수 (기능별)

| 변수 | 용도 | 비고 |
|------|------|------|
| `BANANA_API_KEY`, `BANANA_MODEL_KEY` | Pipeline V2 키프레임 실제 이미지 (Banana.dev) | 없으면 키프레임은 플레이스홀더 PNG |
| `VEO_API_KEY`, `VEO_PROJECT_ID`, `VEO_LOCATION` | Pipeline V2 실제 비디오 (Veo 3.1) | 현재 진입점은 기존 플로우; V2 전환 후 필요 |
| `PORT` | Backend 포트 | 기본 3000 |
| `STORAGE_TYPE`, `STORAGE_PATH` | 영상 저장 위치 | 기본 local, `./storage/videos` |
| `WORKER_CONCURRENCY` | 동시 Job 수 | 기본 2 |

## 6. 트러블슈팅

| 현상 | 확인·조치 |
|------|------------|
| `Cannot connect to Redis` | `redis-cli ping` → Redis 기동 (`brew services start redis` 또는 `docker run ... redis`) |
| `Cannot find module '@google/generative-ai'` | `packages/worker`에서 `npm install @google/generative-ai fluent-ffmpeg` 실행 |
| `GEMINI_API_KEY` 관련 에러 | `.env`에 유효한 `GEMINI_API_KEY` 설정 |
| 포트 3000 사용 중 | `lsof -ti:3000 \| xargs kill` 또는 `.env`에서 `PORT=3001` 등으로 변경 |
| DB 마이그레이션 오류 | `packages/backend`에서 `npx prisma db push` 후 `npx prisma generate` |
| Frontend "Network Error" | Backend 기동 여부 확인: `curl http://localhost:3000/health` |
