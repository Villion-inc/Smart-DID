## 테스트용 환경변수

지금까지 작업한 Backend + Worker + 콜백·안전검증·GeminiProvider 등을 로컬에서 돌리기 위해 필요한 환경변수입니다.  
**위치:** `library-did-main/.env` (루트에 하나 두면 backend/worker가 공통으로 참조)

### 필수 (동작에 반드시 필요)

| 변수명 | 용도 | 예시 |
|--------|------|------|
| `DATABASE_URL` | Backend Prisma DB | `file:./packages/backend/prisma/dev.db` (SQLite) |
| `JWT_SECRET` | Backend 로그인·인증 | 32자 이상 문자열 |
| `INTERNAL_API_SECRET` | Worker → Backend 콜백 인증 (없으면 JWT_SECRET 사용) | JWT_SECRET과 동일해도 됨 |
| `REDIS_HOST` | BullMQ 큐 | `localhost` |
| `REDIS_PORT` | Redis 포트 | `6379` |
| `GEMINI_API_KEY` | Worker(GeminiClient 등) — **없으면 Worker 기동 시 에러** | Google AI Studio에서 발급 |
| `BACKEND_URL` | Worker가 콜백 호출할 Backend 주소 | `http://localhost:3000` |

### 선택 (기능별)

| 변수명 | 용도 | 비고 |
|--------|------|------|
| `BANANA_API_KEY`, `BANANA_MODEL_KEY` | Pipeline V2 키프레임 실제 이미지 (Banana.dev) | 없으면 키프레임은 플레이스홀더 PNG |
| `VEO_API_KEY` | Pipeline V2 실제 비디오 생성 (Veo 3.1) | 없으면 V2 비디오 단계에서 에러 (현재 진입점은 기존 플로우) |
| `VEO_PROJECT_ID`, `VEO_LOCATION` | Veo Vertex AI 프로젝트/리전 | 기본값: your-project-id, us-central1 |
| `PORT` | Backend 포트 | 기본 3000 |
| `STORAGE_TYPE`, `STORAGE_PATH` | 영상 저장 위치 | 기본 local, `./storage/videos` |
| `WORKER_CONCURRENCY` | 동시 Job 수 | 기본 2 |

### 현재 테스트 가능 범위 (npm run dev 기준)

지금 로컬에서 `npm run dev`로 돌렸을 때 **어디까지 실제로 동작하는지** 정리한 내용입니다.

| 구분 | 구현 여부 | 비고 |
|------|------------|------|
| **Frontend** | ✅ 동작 | 로그인, 도서 검색, 상세, 영상 생성 요청, 상태 표시. Backend API 연동. |
| **Backend API** | ✅ 동작 | 인증, 도서 검색/상세, 영상 요청(큐 등록), DB(Prisma/SQLite), Swagger 문서. |
| **영상 요청 → 큐 등록** | ✅ 동작 | 요청 시 상태 `QUEUED`로 저장, BullMQ에 Job 추가. |
| **Worker Job 처리** | ✅ 동작 | Job 수신 → 안전 검증(실제) → 장면 프롬프트 생성(실제) → **장면 생성·병합·자막은 Mock**. |
| **Worker → Backend 콜백** | ✅ 동작 | 완료 시 `READY` + videoUrl/subtitleUrl, 실패 시 `FAILED` + errorMessage 로 DB 갱신. |
| **상태 흐름** | ⚠️ 부분 | `QUEUED` → (Worker 완료 시) `READY` 또는 `FAILED`. **GENERATING**으로 바꾸는 “작업 시작” 콜백은 미구현이라 화면에서는 QUEUED로 있다가 곧바로 READY/FAILED로 전환됨. |
| **실제 영상 생성** | ❌ Mock | Veo 호출 대신 2초 지연 후 가짜 URL 반환. `mergeScenes`는 Mock 버퍼만 저장. 자막은 실제 VTT 생성·저장. |
| **Pipeline V2** | ❌ 미연동 | Worker 진입점은 여전히 `videoGeneratorService`(기존 플로우). PipelineOrchestratorV2·GeminiProvider는 아직 진입점에서 호출 안 함. |
| **저장** | ✅ 로컬 | `STORAGE_TYPE=local` 기준 로컬 디스크에 영상·자막 파일 저장. S3는 스텁. |

**정리:**  
- **E2E로 확인 가능한 것:** “영상 생성 요청 → 큐 등록 → Worker가 Mock으로 처리 → Backend DB가 READY/FAILED로 갱신 → 프론트에서 상태·URL 확인”까지.  
- **실제 AI/영상:** Veo·키프레임·FFmpeg 병합은 아직 Mock 또는 미연동. Pipeline V2 전환 후 실제 연동 예정.

### 테스트 전 준비

1. **Redis 실행**  
   `redis-server` 또는 Docker: `docker run -d -p 6379:6379 redis`

2. **DB 마이그레이션**  
   `library-did-main` 루트에서:
   ```bash
   cd packages/backend && npx prisma migrate deploy
   # 또는 스키마만 반영: npx prisma db push
   npx prisma generate
   ```

3. **의존성**  
   worker에서: `npm install @google/generative-ai axios fluent-ffmpeg`  
   (이미 package.json에 있으면 생략)

4. **실행 순서**  
   Backend → Worker 순으로 기동. Frontend는 `REACT_APP_API_URL=http://localhost:3000/api` 로 Backend 지정.
