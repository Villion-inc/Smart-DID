# 아산도서관 Smart DID — 연동 및 구현 현황 분석

**기준 문서:** `아산도서관_개발_Final.md`  
**분석 대상:** `library-did-main`(백엔드·프론트), `library-trailer-engine-main`(AI 영상 생성 엔진)  
**분석일:** 2026-02-09

---

## 1. 두 프로젝트 연동 여부

### 결론: **현재 연동되어 있지 않습니다.**

| 항목 | 상태 | 설명 |
|------|------|------|
| HTTP 연동 | ❌ 없음 | lib-mvp Worker가 trailer-engine의 `POST /video-engine`을 호출하지 않음 |
| 코드 통합 | ❌ 없음 | trailer-engine 코드가 lib-mvp 패키지에 복사/임포트되지 않음 |
| 공유 큐/메시지 | ❌ 없음 | 서로 다른 프로세스로 독립 실행만 가능 |
| 공유 타입 | ❌ 없음 | trailer-engine 타입과 lib-mvp `@smart-did/shared` 타입이 병합되지 않음 |

- **library-did-main (lib-mvp)**  
  - Worker는 **자체 Mock 파이프라인**만 사용합니다.  
  - `video-generator.service.ts` → `veo.service.ts`(Mock), `prompt.service.ts`(단순 템플릿), `storage.service.ts`(로컬 파일).  
  - trailer-engine을 호출하는 코드나 설정(URL, 환경변수)이 없습니다.

- **library-trailer-engine-main**  
  - 독립 Express 서버(`POST /video-engine`, `GET /health`)로만 동작합니다.  
  - 문서에 나온 **V2 파이프라인**(PipelineOrchestratorV2, Book Grounding, QC, 계층적 재시도)은 `orchestrator.ts` 안에만 있고, **라우트는 `VideoEngine`(구버전, 더미 비디오 생성)**을 사용합니다.  
  - 즉, 문서상 “프로덕션 권장” V2가 HTTP API에는 연결되어 있지 않습니다.

## 2. 문서 대비 구현 현황 (lib-mvp 기준)

### 2.1 문서 1.2절 “컴포넌트별 완성도”와의 대응

| 컴포넌트 | 문서 완성도 | 실제 상태 | 비고 |
|----------|-------------|-----------|------|
| Frontend (React) | 85% | ✅ 유사 | DID/검색/영상 요청 UI 존재, Backend 연동 |
| Shared Types | 90% | ✅ 유사 | video.types, VideoJobData 등 정의됨. **trailer-engine 타입과는 미병합** |
| Backend API | 70% | ⚠️ 부분 | API·라우트·Prisma 구조 있음. **Worker 완료 시 DB 갱신 없음** |
| Worker (영상 생성) | 15% (MOCK) | ✅ 문서와 일치 | Veo/Sora 미연동, FFmpeg 미구현, **완료 시 Backend 콜백 없음** |
| trailer-engine | 80% | ⚠️ 독립만 | 코드는 존재하나 lib-mvp와 **연동/통합 0%** |
| Docker/배포 | 60% | ⚠️ 부분 | Dockerfile 등 있음, 시크릿/모니터링은 미구현 |

### 2.2 문서 4절 “미구현 항목” 대비

#### Critical (MVP 차단) — 문서 4.1

| 항목 | 문서 | 현재 구현 |
|------|------|-----------|
| trailer-engine 통합 | Worker에 PipelineOrchestratorV2 통합 | ❌ 미구현. Worker는 자체 Mock만 사용 |
| FFmpeg 영상 병합 | fluent-ffmpeg 연동 | ❌ `mergeScenes()`는 Mock 버퍼만 저장 |
| 프로덕션 DB | PostgreSQL + Prisma | ⚠️ Prisma 스키마·리포지토리 있음. `db/index.ts`의 InMemoryDB는 테스트용으로 보임 |
| 콘텐츠 안전 검증 | QC safetyGate 통합 | ❌ `validateScene()` 항상 true, safetyGate 미연동 |

#### Important — 문서 4.2

| 항목 | 현재 구현 |
|------|-----------|
| S3 스토리지 | ❌ `storage.service.ts`는 로컬 파일만. S3/ presigned URL 없음 |
| ALPAS 실제 API | ⚠️ `alpas.service.ts`에 AD201/AD206/AE117 연동 구현됨. `alpas-real.service.ts`는 미사용 스켈레톤 |
| Prisma 마이그레이션 | ⚠️ `prisma/schema.prisma`, migrations 폴더 존재. 실제 DB 연결 여부는 환경 의존 |
| WebSocket 실시간 알림 | ❌ 미구현 |

#### 추가로 확인된 중요 결함

- **Worker 완료/실패 시 Backend DB 갱신 없음**  
  - Worker는 BullMQ `completed`/`failed` 이벤트에서 **로깅만** 하고,  
    `videoRepository.update(bookId, { status: 'READY'|'FAILED', videoUrl })` 같은 콜백을 하지 않습니다.  
  - 따라서 큐에 넣어도 상태가 QUEUED에서 GENERATING으로 바뀌지 않고, 완료해도 READY/FAILED로 남지 않습니다.  
- **작업 시작 시 GENERATING 설정 없음**  
  - Backend에 “작업 시작 시 status를 GENERATING으로 바꾸는” 로직이 없습니다. (Worker가 Backend를 호출하지 않음)

## 3. trailer-engine 쪽 현황 (문서 대비)

- **문서 2절 (V2 파이프라인)**  
  - `src/pipeline/orchestrator.ts`에 **PipelineOrchestratorV2** 존재.  
  - Book Grounding, Style Bible, 계층적 재시도, QC, 자막 등 문서와 유사한 단계가 코드에 있음.  
- **HTTP API와의 불일치**  
  - `POST /video-engine`은 **VideoEngine** (`src/services/video-engine.ts`)만 사용.  
  - VideoEngine은 구버전 Gemini + **createDummyVideo** 기반이라, 문서에 적힌 “Sora 2/Veo 3.1, Keyframe, QC” V2 파이프라인과는 다릅니다.  
- **문서 9.3 “테스트 명령”**  
  - `npm run generate:v2` → `package.json`에는 `"generate:v2":"ts-node src/test/generate-complete-v2.ts"` 있음.  
  - 실제 파일은 `src/test/generate-v2-pipeline.ts` 등으로 존재. CLI로 V2 파이프라인 동작 확인은 가능한 구조이나, **서버 라우트는 여전히 V1(VideoEngine)** 입니다.

## 4. 요약 표

| 구분 | 내용 |
|------|------|
| **연동** | library-did-main ↔ library-trailer-engine-main **연동 없음** (HTTP/코드/타입 모두 미연결) |
| **문서 구현도** | 문서의 “Worker 15% Mock, trailer-engine 80%”와 **대체로 일치**. 다만 trailer-engine 80%는 “독립 실행 기준”이고, lib-mvp와의 통합은 **0%** |
| **MVP 차단 이슈** | (1) trailer-engine 미통합 (2) FFmpeg 미구현 (3) **Worker → Backend 콜백 미구현**(완료/실패 시 DB 갱신 없음) (4) QC/safetyGate 미연동 |
| **권장 다음 단계** | 문서 3절 “통합 계획”대로 Worker에 PipelineOrchestratorV2 통합 + **Worker 완료/실패 시 Backend API 콜백** 구현 + FFmpeg·Storage(S3)·QC 연동 |

이 문서는 `아산도서관_개발_Final.md`와 두 레포지토리 코드를 기준으로 한 연동·구현 현황 정리입니다. 배포/DB 연결 등 환경별 차이는 별도 확인이 필요합니다.
